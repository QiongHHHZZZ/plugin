# 变更提案: UI 文案硬编码汉化

## 需求背景
当前插件 UI 与聊天提示多为英文，影响中文用户使用。需要在不引入 i18n 框架的前提下，将用户可见文案直接替换为中文。

## 变更内容
1. 将配置窗口中的按钮、提示、表头、分区标题等文案替换为中文。
2. 将聊天栏提示与错误信息替换为中文，保持功能行为不变。
3. 保留 ImGui 内部 ID（###/## 后缀）与内部标识不变，避免影响交互状态。

## 影响范围
- 模块: gui, core, layout, memory
- 文件: ReMakePlacePlugin/Gui/ConfigurationWindow.cs, ReMakePlacePlugin/ReMakePlacePlugin.cs, ReMakePlacePlugin/SaveLayoutManager.cs, ReMakePlacePlugin/Memory.cs
- API: 无
- 数据: 无

## 核心场景

### 需求: UI 文案汉化
**模块:** gui
将配置窗口与列表的用户可见文本替换为中文。

#### 场景: 配置窗口与列表
用户打开配置窗口并进行保存/加载/染色等操作
- 预期结果: 按钮、表头、提示均为中文且可读
- 预期结果: 功能与交互逻辑不变

### 需求: 聊天日志汉化
**模块:** core/layout/memory
将聊天栏提示与错误信息替换为中文。

#### 场景: 执行保存/加载/染色
用户执行保存/加载/染色流程
- 预期结果: 聊天栏提示为中文
- 预期结果: 错误提示仍能清晰表达原因

## 风险评估
- 风险: 中文文案长度增加导致按钮或表头拥挤
- 缓解: 选择短句/缩写，必要时调整文本措辞
