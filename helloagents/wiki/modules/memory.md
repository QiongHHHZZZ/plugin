# 内存与状态访问

## 目的
封装对游戏内存结构的访问，提供模式判断与位置/旋转写入能力。

## 模块概述
- **职责:** 读取 HousingModule/LayoutWorld，提供对象列表、模式判断、位置/旋转写入
- **状态:** ✅稳定
- **最后更新:** 2026-01-20

## 规范

### 需求: 模式与对象访问
**模块:** memory
提供 IsHousingMode/CanEditItem/CanDyeItem 等接口供 UI 与核心逻辑判断。

#### 场景: UI 操作前检查
- 预期结果1: 返回当前模式可用性
- 预期结果2: 避免在无效状态下进行写入

### 需求: 模式校验恒真
**模块:** memory
将 IsHousingMode/CanEditItem/CanDyeItem 固定为 true，绕过模式限制。

#### 场景: 非布局模式触发操作
- 预期结果1: 不因模式校验而阻断流程
- 预期结果2: 保持调用路径不变

### 需求: 内存错误提示汉化
**模块:** memory
任意摆放内存地址异常时输出中文错误提示。

#### 场景: 任意摆放初始化失败
内存地址初始化失败或缺失
- 预期结果1: 输出中文错误提示
- 预期结果2: 不继续写入内存
