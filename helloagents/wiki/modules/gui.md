# 界面与交互

## 目的
提供配置窗口、文件选择、功能触发与可视化提示。

## 模块概述
- **职责:** 绘制 UI、读取/写入配置、触发保存/加载/染色/应用操作
- **状态:** ✅稳定
- **最后更新:** 2026-01-20

## 规范

### 需求: 配置窗口与操作按钮
**模块:** gui
提供保存/加载/染色等按钮并在必要时给出提示。

#### 场景: 用户在配置窗口点击动作按钮
插件已初始化并打开配置窗口
- 预期结果1: 触发对应的核心操作
- 预期结果2: 不满足模式时给出错误提示

### 需求: UI 文案汉化
**模块:** gui
配置窗口与列表文案使用中文显示。

#### 场景: 打开配置窗口
打开配置窗口进行保存/加载/染色等操作
- 预期结果1: 按钮、提示、表头为中文
- 预期结果2: 交互逻辑不变

### 需求: 图标按钮对齐
**模块:** gui
带图标按钮的图标与文字需保持在按钮框内对齐显示。

#### 场景: 列表工具栏显示刷新按钮
显示带图标的刷新按钮
- 预期结果1: 图标与文字位于按钮背景内
- 预期结果2: 视觉对齐居中

### 需求: 设置按钮列宽适配
**模块:** gui
列表中的设置按钮需完整显示且不遮挡物品列。

#### 场景: 列表显示设置按钮
显示带“设置”按钮的列表
- 预期结果1: 设置按钮完整显示
- 预期结果2: 不覆盖物品图标与名称
